## Test matrix

| Scenario | What to verify |
| --- | --- |
| Successful checkout | Widget reaches success and `payment.captured` arrives. |
| Terms required | Terms screen appears before hosted card entry. |
| Duplicate create call | Same order ID does not create duplicate payment sessions. |
| Redirect validation | Unconfigured redirect origin is rejected. |
| Failed payment | Customer sees failure and webhook is handled idempotently. |
| Expired/stuck session | Store can show retry and reconcile state. |
| Webhook duplicate | Duplicate event does not double-fulfill order. |
| Webhook bad signature | Endpoint rejects the event. |
| Refund/reversal | Operational flow is tested before launch. |
| Payout destination | Withdrawal is blocked until destination is approved. |

## Manual end-to-end test

1. Create a checkout session from your backend.
2. Mount the widget.
3. Complete card entry and 3DS if requested.
4. Confirm the widget reaches success.
5. Confirm your webhook endpoint receives `payment.captured`.
6. Confirm your order is marked paid once.
7. Confirm the payment appears in the merchant dashboard.
8. Confirm a voucher reference is visible when issued.

## Internal smoke command

Repository maintainers can run:

```bash
pnpm smoke:release
```

This is an internal release smoke check, not a merchant integration requirement.
