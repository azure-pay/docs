---
title: "Mount the checkout widget"
description: "Render VoucherPay checkout in the browser with session ID and client secret."
---

The widget is browser-safe. It must receive only:

- `sessionId`
- `clientSecret`
- optional public API base URL

Never pass your merchant API key to the browser.

## Install

```bash
npm install @azure-pay/checkout-widget
```

## Basic mount

```html
<div id="voucherpay-checkout"></div>
```

```ts
import { mount } from "@azure-pay/checkout-widget";

const handle = mount({
  target: "#voucherpay-checkout",
  sessionId: "cm...",
  clientSecret: "cm..._secret_...",
  apiBaseUrl: "https://api-demo-02f5.up.railway.app",
});
```

## Options

| Option | Required | Notes |
| --- | --- | --- |
| `target` | Yes | CSS selector or HTMLElement. |
| `sessionId` | Yes | Returned by your backend. |
| `clientSecret` | Yes | Returned by your backend. |
| `apiBaseUrl` | No | Defaults to production API when omitted. |
| `cardholderEmail` | No | Captured with terms acceptance evidence. |
| `locale` | No | Captured with terms acceptance evidence. |
| `pollIntervalMs` | No | Forces fixed polling interval. |
| `maxPollDurationMs` | No | Defaults to 15 minutes. |

The `iframeUrl` option is deprecated. The widget discovers the current hosted-payment URL from VoucherPay status and start-payment responses.

## Lifecycle handlers

```ts
const handle = mount({
  target: "#voucherpay-checkout",
  sessionId,
  clientSecret,
  onReady: () => {
    console.log("Checkout ready");
  },
  onTermsRequired: ({ termsVersionId }) => {
    console.log("Terms required", termsVersionId);
  },
  onTermsAccepted: () => {
    console.log("Terms accepted");
  },
  onProcessing: () => {
    console.log("Payment processing");
  },
  onSuccess: ({ paymentSessionId, voucherReference }) => {
    console.log("Captured", paymentSessionId, voucherReference);
  },
  onError: ({ code, message }) => {
    console.error(code, message);
  },
});
```

## Polling behavior

By default, the widget follows a backoff schedule:

```text
5s -> 10s -> 20s -> 30s -> 60s
```

It continues at the last interval until the maximum polling duration is reached. Set `pollIntervalMs` only when you intentionally want fixed polling.

## Cleanup

Destroy the widget when the checkout component unmounts:

```ts
handle.destroy();
```

This prevents duplicate polling loops and duplicate hosted-payment frames after route changes or React remounts.
