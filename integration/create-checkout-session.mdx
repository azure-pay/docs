---
title: "Create checkout sessions"
description: "API reference and examples for creating VoucherPay checkout sessions."
---

Create checkout sessions from your backend. This binds the order to your merchant account before the cardholder reaches card entry.

## Endpoint

```http
POST https://api-demo-02f5.up.railway.app/v1/public/checkout-sessions
Authorization: Bearer vpay_test_...
Content-Type: application/json
Idempotency-Key: ORDER-1234
```

## Request body

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `externalReference` | string | Yes | Your immutable order reference. Max 80 chars. |
| `amountCents` | integer | Yes | Positive integer in minor units. |
| `currency` | string | Yes | Currently `EUR` only. |
| `customerEmail` | string | No | Cardholder email when known. |
| `productTitle` | string | No | Shown in acquirer/checkout context. Max 120 chars. |
| `successRedirect` | URL | No | HTTPS return URL after success. |
| `failureRedirect` | URL | No | HTTPS return URL after failure. |
| `cancelRedirect` | URL | No | HTTPS return URL after cancellation. |
| `inProgressRedirect` | URL | No | HTTPS return URL if payment remains in progress. |
| `metadata` | object | Yes | Key-value metadata object. Use `{}` if empty. |

## cURL example

```bash
curl https://api-demo-02f5.up.railway.app/v1/public/checkout-sessions \
  -H "Authorization: Bearer $VOUCHERPAY_SECRET_KEY" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: ORDER-1234" \
  --data '{
    "externalReference": "ORDER-1234",
    "amountCents": 9900,
    "currency": "EUR",
    "customerEmail": "customer@example.com",
    "productTitle": "Order ORDER-1234",
    "successRedirect": "https://store.example.com/thanks",
    "failureRedirect": "https://store.example.com/checkout?failed=1",
    "cancelRedirect": "https://store.example.com/checkout?cancelled=1",
    "inProgressRedirect": "https://store.example.com/checkout?pending=1",
    "metadata": {
      "orderId": "ORDER-1234"
    }
  }'
```

## SDK example

```ts
import { VoucherPay } from "@voucherpay/server";

const voucherpay = new VoucherPay(process.env.VOUCHERPAY_SECRET_KEY!, {
  apiBaseUrl: "https://api-demo-02f5.up.railway.app",
});

const session = await voucherpay.sessions.create(
  {
    externalReference: "ORDER-1234",
    amountCents: 9900,
    currency: "EUR",
    customerEmail: "customer@example.com",
    productTitle: "Order ORDER-1234",
    successRedirect: "https://store.example.com/thanks",
    failureRedirect: "https://store.example.com/checkout?failed=1",
    cancelRedirect: "https://store.example.com/checkout?cancelled=1",
    inProgressRedirect: "https://store.example.com/checkout?pending=1",
    metadata: {
      orderId: "ORDER-1234",
    },
  },
  {
    idempotencyKey: "ORDER-1234",
  },
);
```

## Redirect validation

VoucherPay rejects redirect URLs when:

- the URL is unsafe or private-network;
- the merchant has no allowed origins configured;
- the URL origin is not in the merchant allowed origins.

Configure origins before sending redirect URLs.

## Duplicate requests

VoucherPay deduplicates checkout sessions by merchant and `externalReference`. Use the same `Idempotency-Key` when retrying the same operation.
