---
title: "Go-live checklist"
description: "Production readiness checklist for launching VoucherPay on a merchant store."
---

Do not go live until every item is complete.

## Merchant account

- KYB approved.
- Acquirer provisioning completed.
- Production merchant API key issued.
- Webhook signing secret issued.
- Payout destination approved.
- Allowed production origins configured.
- Terms state confirmed.
- Cardholder KYC disabled unless VoucherPay confirms it is live.

## Store integration

- Checkout sessions are created server-side.
- Amount and currency come from your database, not the browser.
- `EUR` is used unless VoucherPay has enabled more currencies for your account.
- Idempotency keys are stable and derived from order/operation IDs.
- Widget receives only `sessionId` and `clientSecret`.
- Redirect URLs are HTTPS and allowed.
- Webhook endpoint verifies `VoucherPay-Signature`.
- Duplicate webhooks do not double-fulfill orders.
- Payment fulfillment depends on webhook/reconciliation, not redirects alone.

## Testing

- Successful checkout tested.
- Terms-required checkout tested if enabled.
- Failed payment tested.
- Duplicate checkout create retry tested.
- Webhook signature failure tested.
- Duplicate webhook delivery tested.
- Refund/reversal process tested in sandbox or staging.
- Payout/withdrawal operational path tested.

## Monitoring

Monitor:

- checkout create API failures;
- webhook delivery failures;
- payment sessions stuck in processing;
- failed refunds or reversals;
- unavailable payout destinations;
- support tickets tied to payment session IDs.

## Launch day

1. Rotate from demo/test keys to production keys.
2. Confirm production webhook endpoint is active.
3. Run a low-value live smoke transaction if permitted.
4. Confirm `payment.captured` webhook is received.
5. Confirm the order is marked paid once.
6. Confirm payment and voucher appear in the merchant portal.
7. Keep support and engineering available during first live traffic.
