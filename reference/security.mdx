---
title: "Security"
description: "Security requirements for a safe VoucherPay integration."
---

## Keep merchant API keys server-side

Correct:

```text
Browser -> merchant backend -> VoucherPay
```

Incorrect:

```text
Browser -> VoucherPay with merchant API key
```

The browser should receive only `sessionId` and `clientSecret`.

## Do not trust client totals

Never let the browser choose:

- amount;
- currency;
- order reference;
- merchant ID;
- product price.

Load the order from your database, then create the VoucherPay session server-side.

## Verify webhooks

Always verify `VoucherPay-Signature` against the raw request body. Reject events with invalid signatures or stale timestamps.

## Use HTTPS

Use HTTPS for:

- storefront checkout pages;
- redirect URLs;
- webhook endpoints;
- dashboard access.

## Hosted card entry and PCI scope

VoucherPay uses hosted card entry through the acquiring provider. Your store should not collect raw card numbers, expiry dates, or CVC values.

## Logging rules

Do not log:

- full API keys;
- webhook signing secrets;
- full client secrets;
- cardholder card data.

Safe identifiers to log:

- your `externalReference`;
- VoucherPay session ID;
- webhook event ID;
- delivery ID;
- voucher reference.
