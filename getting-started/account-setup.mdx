Before your store can create checkout sessions, the merchant account must be operational.

## Setup checklist

1. Receive merchant dashboard access.
2. Complete merchant KYB.
3. Wait for VoucherPay approval.
4. Configure allowed storefront origins.
5. Create a merchant API key.
6. Configure a webhook endpoint and signing secret.
7. Confirm payout destination requirements.
8. Confirm whether terms acceptance is enabled for your merchant.

## API keys

API keys start with `vpay_` and authenticate your backend to VoucherPay.

Use them only on your server.

Do not put API keys in:

- browser JavaScript;
- mobile apps;
- GitHub;
- analytics events;
- screenshots;
- support tickets.

## Allowed origins

If you send redirect URLs when creating a checkout session, their origins must be configured on your merchant account.

Example origin:

```text
https://store.example.com
```

Use only scheme, host, and optional port. Do not include paths.

If no allowed origin is configured and your request includes redirect URLs, VoucherPay rejects the request.

## Terms

Terms acceptance can be required before payment. Terms are managed by VoucherPay for the merchant account today; the merchant portal shows the current terms state but does not self-publish arbitrary merchant terms.

When terms are required, VoucherPay snapshots the current published terms version onto the checkout session and records acceptance evidence.

## Cardholder KYC

Cardholder KYC is not live. Do not enable cardholder KYC for a production merchant until VoucherPay confirms it is available.
