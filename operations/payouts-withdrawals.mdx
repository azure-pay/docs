---
title: "Payouts and withdrawals"
description: "Understand when captured value becomes available and how withdrawals work."
---

Captured payment value can move through several states before it is withdrawable.

## Balance states

| State | Meaning |
| --- | --- |
| Captured | Payment succeeded and voucher issuance can complete. |
| On merchant books | Voucher value has moved to the merchant ledger. |
| Held or locked | Value is blocked by settlement delay, reserve, risk review, chargeback, or missing approval. |
| Available | Value can be included in a withdrawal request. |
| Withdrawn | Withdrawal has completed. |

## Payout destinations

Withdrawals require an approved payout destination. Depending on account setup, this can include bank or wallet destinations.

If no destination is active, the merchant cannot withdraw even if value is available.

## Withdrawal flow

1. Merchant chooses an active payout destination.
2. Merchant selects available value.
3. Merchant submits the withdrawal request.
4. VoucherPay reviews and processes the request.
5. VoucherPay marks the withdrawal completed or rejected.
6. Webhook events and portal state update when enabled.

## Why value can be locked

Value can be unavailable because of:

- settlement delay;
- rolling reserve;
- chargeback hold;
- pending payout destination approval;
- operational review;
- account restrictions.

Your commercial agreement and account configuration determine which controls apply in production.
